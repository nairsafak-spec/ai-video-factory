# Architecture Decision Records (ADR)

This folder contains all Architecture Decision Records for the project. Each ADR captures a significant architectural or technology decision — its context, the alternatives considered, the decision made, and its consequences.

## About This Folder

- Every ADR lives here as its own file, named `ADR-NNN-short-title.md` (e.g. `ADR-001-choose-vector-database.md`).
- ADRs are numbered sequentially and never reused, even when rejected or deprecated.
- Records are **immutable once Accepted** — to change direction, add a new ADR that supersedes the old one.

## Conventions

- **Numbering, status values, required sections, and the ADR template** are defined in the root [`DECISION_LOG.md`](../../DECISION_LOG.md), which also serves as the master index.
- The process for producing the evidence behind each decision is defined in [`RESEARCH_METHODOLOGY.md`](../../RESEARCH_METHODOLOGY.md).
- Scoring of candidates follows [`EVALUATION_CRITERIA.md`](../../EVALUATION_CRITERIA.md).
